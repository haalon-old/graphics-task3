Зайцев Игорь Олегович, 324 группа

Операционная система: Windows 10 x64 (с cygwin) и Ubuntu 14.04
Оборудование: Intel Core i7 4 ядра по 2 потока, 16 ГБ памяти

Формат файла настроек:
obj-file cube.obj 					имя obj файла для работы
width 1080							ширина итогового изображения
height 1080							и его высота
wifi -2 2.6 2 						позиция роутера
camera-pos 0 -12 -8 				позиция камеры
view-angle-x 90 					угол обзора по горизонтали //view-angle-y is calculated automatically
up 0 -0.5 1 						направление вверх у камеры. желательно чтобы оно было ортогонально направлению обзора камеры //right is calculated automatically
viewdir 0 1 0.5 					направление обзора камеры 
wifi-pow 5000 						мощность wifi сигнала
wifi-radius 0.25					радиус излучателя на итоговом изображении. Он обозначается синей сферой. по умолчанию - 0.5
voxel-res 400 						задает минимальную размерность воксельной сетки. Т.е при voxel-res 400 размерность может быть 1200 400 1000
alpha 0.01							альфа-составляющая. в идеале - мала
gamma 0.5							по-умолчанию 0.5. Отвечает за степень убывания цвета от мощности вокселя		
outside 1							если не 0 - камера видит сквозь первую плоскость что она пересекает. Удобно если у комнаты стены и крыша из одной плоскости
iterations 400000					сколько лучей будет выпущено. Чем больше voxel-res, тем больше итераций надо чтобы хорошо заполнить воксельную сетку
AA-level 5							Анти-алиасинг. Сколько лучей на каждый пиксель. По умолчанию - 1.
filter-radius 2						радиус box-фильтра, по умолчанию 1

Время работы программы: 5 секунд (room_side_fast.txt); 20 секунд (room_top.txt); 1.5 минуты (room_sidest_side.txt)

Бонусы:
анти-алиасинг
параллелизм на openMP
пересечение с треугольником(без него не сделать базу толком)
